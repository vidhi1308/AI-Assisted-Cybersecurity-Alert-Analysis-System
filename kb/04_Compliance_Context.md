# NIST Context for Playbook Framing

## Purpose
Provide conceptual alignment with industry-accepted incident response standards.

This is NOT certification mapping.

---

# Incident Handling Phases (Conceptual)

1. Preparation
2. Detection & Analysis
3. Containment
4. Eradication
5. Recovery
6. Post-Incident Activity

Playbooks should explicitly state which phase they are in.

---

# Safe Language for Compliance

Use:
- "Aligns with NIST Detection & Analysis phase"
- "Supports Respond function under NIST CSF"
- "Contributes to Recover phase"

Do NOT claim:
- "This satisfies NIST control XYZ"

---

# Documentation Expectations

Recommend:
- Timeline capture
- Query results logging
- Action documentation
- Escalation records

---

# Preventive Control Suggestions (General)

- MFA enforcement
- Least privilege review
- Conditional access policies
- Monitoring improvements
- Detection tuning
- Logging expansion
